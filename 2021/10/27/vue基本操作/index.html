基本注意事项

基本操作

切目录

git clone下来之后，cd 进项目文件夹，或者可以右键选择项目文件夹，选择用VSCODE打开

这时候你的根目录应该是你的项目文件夹，目录结构类似：

安装依赖：npm install

确保目录是对了之后，用npm install下载node_modules文件夹里的依赖。node_modules的文件夹很多，是不会传到git仓库去的，所以clone下来必定要npm install。除此之外，项目进行过程中可能需要import新的包，此时也是要npm install更新依赖的。一般项目启动不起来而且代码本身没错，基本都是没有安装依赖的问题。

这种报错都不用担心，只要是warn不是error，npm run serve跑得起来就都没问题

运行代码：npm run serve

如图所示就是正确把项目启动了，可以在浏览器上看到效果

项目打包: npm run build

运行完这个代码之后会生成一个dist文件夹，dist底下是一个叫做static的文件夹，把这个static文件夹拷到后端文件夹中resources目录下就可以。除此之外还会有一个static外面有个index.html的文件，也得拷贝过去，而且得拷贝到static里面

打包后的结构

放到SpringBoot后端的结构（注意index.html被移到static里面去了）

项目结构

assets

存放一些全局css，在img文件夹下存放网页要用的图片

component

存放一些公共组件

plugins

一些配置，比如这里封装好了axios和element-ui，在其他网页都可以直接用

router

很重要，配置网页路径用的，每新写一个页面就要那里注册路由，这样才能实现页面间的跳转。写法可以参考已经写好的代码

store

存放全局通用变量、方法，可以在所有页面中通用。具体使用方法可以看一下Login.vue里怎么写的。

utils

一般会放一些很杂的工具，比如我们写过一个问卷星的项目，往里面存了网页生成pdf的代码。

views

核心文件夹，所有页面都写在这里。关于页面里每一段都是在干啥的，建议自己参悟一下，应该不算难。另外最好自己看看vue页面的生命周期，知道一下created，mounted这些函数是干啥的。一个页面基本结构如下：

关于前后端连接

baseURL

在src/plugins/axios.js里更改baseURL，确保baseURL的值是后端ip+端口

如果要部署到服务器上的话，这里的baseURL一定要记得改。前端和后端这里不大一样，后端写成localhost是ok的，因为后端它自己就跑在服务器上，但是前端是跑在用户自己的机子的浏览器上的，还是得写服务器的地址。

关于前端端口号和后端不一致的问题

在本地跑的时候，前端后端分别运行，是没法同时占用一个端口的，所以要写成两个端口。后端有相应的代码处理跨域问题。如果想修改前端的端口号，找到vue.config.js文件修改就好

最后前端打包成dist之后，网页的运行端口是以后端为准的。后端是localhost:8080的话，最后打开页面就是例如localhost:8080/login。前端这个端口只是为了让端口不冲突，开发方便而设计的。

一些很杂的注意事项（想到哪写到哪了属于是）

  • 凡是函数里出现await的，函数前面都必须加上async修饰，代表这是异步函数

  • 每个vue文件，<template></template>之间只准包裹一对标签

    • 一个页面里用到的变量写在data(){return{}}里。要注意的是，<template></template>包裹的html使用这些变量的时候是直接使用的，但是<script></script>里要引用他们，前面必须加上一个this.。调用函数也是如此

  • 好习惯是，<style scoped></style>都加上这个scoped，这样可以限制css只用于当前的页面，不会因为和其它页面同名而冲突。
