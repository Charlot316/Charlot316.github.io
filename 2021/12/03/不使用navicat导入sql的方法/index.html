不使用navicat导入sql的方法

给队友们写的指南

Posted by Charlot on 2021-12-03

IDEA 自带的Database插件

点击这里打开Database插件，设置数据源（没有的话，去plugins里下载就可以）

会跳出设置数据库信息的窗口

报这个错的时候按照这个网页提供的方法改正就可以（其实就是点进Set time zone，把时区修改为GMT，像下图这样）

接下来，右键你刚才创建的和数据库的连接，选择new schema

命名为academic_system

右键academic_system，选择run sql script

选择提供的sql并执行即可

执行成功后，你就能看到新建的表格啦

之前提供的那个sql里面，user的有几列被设置为了not null，这是有问题的，需要去掉。（只是这一版sql有问题而已！之后我会提供正确的sql，可以直接使用正确的sql进行加载，就不用做后面的操作了）

右键选中user表，选择modify table，双击选中需要修改的几个属性把not null的勾去掉即可（需要修改的属性有:image、like_num、follow_num）

（like和follow_num也是同样的操作，我就不演示了）

修改完之后点击execute即可

此时后端就是完全可以使用的了，启动后端之后，前端的登陆注册功能就都可以使用啦！

另外附赠一下将数据库导出为sql的操作。

navicat是右键要导出的数据库，选择转储SQL文件，选择导出结构和数据或选择仅结构（顾名思义就知道两个选项分别是什么意思）

idea是这样：

点击之后底下就会生成所有表格的sql，点击保存即可

PS：我个人认为navicat生成的sql更好，因为navicat会有DROP TABLE IF EXISTS application;这句话，IDEA是没有的，所以运行navicat生成的sql可以直接更新数据库结构，而IDEA生成的sql只能用于新建数据库结构，没法更新，因为会提示XXX table already exists。不过不用担心，后端提供的sql一般都是navicat生成的，我们拿来就可以直接用。

注意：前提是你已经安装好mysql，而且已经设置好了root账户和密码，如果root账户和密码没设置的话还需要先设置一下

参考这个网页

直接用控制台加载sql

在cmd输入mysql -u root -p这句话进入mysql，接下来会提示你输入密码，输入123456（你自己设置的）即可

（如果提mysql不是内部或外部命令也不是可运行的话是环境变量的配置问题，请参考这个网页）

新建academic_system数据库，输入指令：

CREATE DATABASE academic_system;

接下来输入source命令

source ${sql文件的存储路径}

到这里为止数据库就建立完成啦！

接下来修改三处not null的bug，输入下列命令

alter table user modify image varchar(100) null;

alter table user modify like_num int null;

alter table user modify follow_num int null;

这样就全部完成啦！前端的登录注册页面就可以使用啦

注：我其实会提供已经改好null这个bug的sql，大家不用自己去改这个bug，这样节省大家一些时间。但是我还是在md里提及了修改null的bug的方法，如果大家以后遇到了类似的问题，可以参考这里的修改方法。
