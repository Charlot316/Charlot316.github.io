Web编程基础

什么是WWW

应用层的编程（socket是传输层）

  • WWW 指万维网(World Wide Web)
  • 万维网也常被称为Web
  • Web是由遍布全球的计算机所组成的网络
  • 所有 Web 中的计算机都可以彼此通信
  • 所有这些计算机都使用名为 HTTP 的通信 标准

WWW如何工作

  • Web信息存储于被称为网页的文档中
  • 网页存储于被称为 Web 服务器的计算机上
  • 读取网页的计算机称为 Web 客户端
  • Web客户端通过浏览器来查看网页
  • 主要的浏览器有
    • Internet Explorer (IE内核)
    • Mozilla Firefox( Firefox 内核)
    • 360( IE内核+Webkit内核双核浏览器) – Google浏览器(webkit内核)

Web特点

  1. Web是易导航和图形化
  2. Web是具有平台无关性
  3. Web是支持分布式结构
  4. Web是具有动态性
  5. Web是具有交互性

Web标准

  • web 规则的制定主体是 W3C
  • W3C 指的是万维网联盟(World Wide Web Consortium)
  • web 标准即W3C 的各种规范
  • 最核心的 web 标准是 HTML、CSS、XML
  • 最新的 HTML 标准是 XHTML 1.0

网站

  • 网站(Website)是指在因特网上根据一定的 规则，使用HTML等工具制作的用于展示特定内容相关网页的集合。
  • 网页:构成网站的基本元素。
  • 首页:也称主页，即一个网站的入口网页， 首页往往会被编辑得易于了解该网站，并引导互联网用户浏览网站其他部分的内容。 这部分内容一般被认为是一个目录性质的内容。大多数作为首页的文件名是index、 default、main或portal加上扩展名

Web服务器

  • Web服务器:指在Internet上提供Web访问服务的站点，是由计算机软件和硬件组成的 有机整体。
  • 必须为Web服务器配置IP地址和域名，才能对外提供Web服务。
  • Web服务器(Server端):用来专门提供WWW服 务的服务器软件称为Web服务器。
  • 常用的Web服务器:
    • Apache
    • IIS
    • Tomcate等

  • 常用的应用服务器:

    • WebLogic
    • WebSphere

    • JBoss等

IP地址、端口号

  • 在网络上，计算机通过IP地址或主机名进行标识， 这样使得位于不同地理位置的计算机有可能互相访问和通信。
    • ip是由32位的数字表示的,通常被分成四部分。 例如:192.168.0.1
  • 本地回路的IP地址: 127.0.0.1 或localhost
    • Port(端口号):逻辑意义上的数据传输通道，或者说模拟通道。
  • 端口号的范围为0~65535之间，0~1023之间的端口数是用于一些知名的网络服务和应用 (我们一般不使用)
    • 例如:Web服务使用端口80,FTP服务使用端口21等

URL

  • 统一资源定位器(Uniform Resource Locator，URL)也被称为网页地址，如同
    在网络上的门牌，是因特网上标准的资源 的地址(Address)。
  • 格式:协议类型://服务器地址(端口号)/ 路径/文件名
    • 默认的端口号可以被省略
    • http://www.sina.com.cn
    • http://www.baidu.com

HTTP协议

  • HTTP协议(Hyper Text Transfer Protocol, 超文本传输协议)
  • HTTP适用于从Web服务器传输HTML文件 到本地浏览器的通信协议。
  • 该协议是基于请求/响应形式的结构(相当于 客户机/服务器结构)。
  • HTTP协议是无状态的协议。
  • 还有一个HTTP的安全版本称为HTTPS。

超链接

  • 指从一个网页指向另一个目标的连接关系，这个 目标可以是另一个网页，也可以是相同网页上的 不同位置，还可以是一个图片、一个电子邮件地 址、一个文件，甚至是一个应用程序。
  • 超链接在本质上属于一个网页的一部分，它是一 种允许我们同其他网页或站点之间进行连接的元素。
  • 例如:<a href="http://baike.baidu.com">百科 </a>

网站构建技术

  • HTML 4.01&HTML5
  • CSS的使用(样式表)
  • XHTML
  • XML和XSLT
  • 客户端脚本
  • 服务器端脚本
  • 通过 SQL 管理数据

HTML

  • 超文本标记语言(HTML)
    • Hyper Text Markup Language
    • 用来描述页面的内容和结构,定义超文本文档的 结构和格式;
    • HTML告诉浏览器如何把内容显示给用户看到;

J2EE架构

体系结构发展

  • 从一层到多层，具有多个层次的体系结构 已经变成了主流的企业级应用的体系结构。

    • C/S 结构 （客户、服务器）
    • B/S结构
    • N层结构
    • 前台、中台和后台

一层体系结构

  • 一层体系结构就是将所有的程序和数据都配置在一个 主框架中的系统，该应用程序中包括了数据访问、数 据表示和业务逻辑。
  • 优点:
    • 构造十分简单
  • 缺点:
    • 故障容易集中在一起，远距离访问的限制;
    • 功能紧紧耦合在一起，代码重用、代码可维护性和代码的可扩展都十分困难。
    • 不是分布式的，不具有可伸缩性。
    • 系统的功能受限于主机的能力，主机一旦出问题，整个系统就无法再正常运行

二层体系结构

  • 二层体系结构就是，从一层体系结构中，将界面显示部分和一部分的业务逻辑向客户端转移。C/S体系结构就是这种体系结构的一个例子。
  • 由于二层体系结构中的界面显示部分和业务逻辑部分的划分不是十分明确，所以界面部分的变更会影响到业务逻辑部分，业务逻辑部分的变更同样也会影响到界面部分 ;
  • 二层体系结构也产生了像客户端更新这类新的问题

传统的客户端/服务器：“胖”客户端

  • 显示、业务逻辑，数据模型的处理在客户端完成，造成功能过渡集中，不利于更新和维护;
    • 每个客户端通过SQL语句访问数据库，数据模型“紧耦合”在每一客户端，数据库结构改变将导致全体客户端改变;
    • 任何一点的更新要对每一客户端进行部署，系统维护成本太大!
  • 每一客户端都需要一个数据库连接，原始数据通过网络传递导致网络拥塞;

三层分布式对象架构

  • 瘦客户端:业务数据模型与UI分开
  • 业务逻辑在中间层，作为“对象服务”。
  • 中间层通过连接池，多线程，并发对象处理等可以处理多个客户端的请求

B/S结构

  • 第一层客户机是用户与整个系统的接口。浏览器将HTML代码转化成图文并茂的网页。网页具备一定的交互功能，允许用户在网页提供的申请表上输入信息提交给后台Web服务器 。
  • 第二层Web服务器将启动相应的进程来响应请求，如果客户机提交的请求包括数据的存取，Web服务器还需与数据库服务器协同完成该处理工作。
  • 第三层数据库服务器的任务类似于C/S模式，负责协调不同的Web服务器发出的SQL请求，管理数据库。
  • Web服务器和数据库服务器通常都可以是分布式的集群部署。

JavaEE体系结构

企业级应用特征

  • 结构复杂，涉及到内外部资源众多、事务密集、 海量数据、多用户、高安全性要求。
  • 采用EAI(Enterprise Applicatoin Integeration, 企业级应用集成)、B2BI(Business to Business Integeration,企业间集成)技术及N层结构。
  • 企业级应用由多个相对独立的小应用组成，并且能够互操作。

为什么需要Java EE

  • 在企业级应用开发中所面临的必须解决的问题:
    • 可部署的、分布式的、可移植的面向Web体系;
    • 远程方法调用、负载平衡、fail over、后端整和、日志和监听以及系统管理;
    • 消息中间件、对象的生命周期的管理、资源池等;
    • 满足企业计算要求
      • 一致性、事务性、安全性
    • 好的特性
      • 可伸缩、可扩展、易维护
  • 为了解决以上所有问题，J2EE提供了一套完 整的框架方案:
    • Java EE提供了一个分布式的、可移植组件 的框架
    • 简化了服务器端n层中间组件的设计
    • Java EE为组件与应用服务器提供标准API
  • 应用服务器容器和组件间的标准合同

JavaEE是什么

  • Java EE是开放的、基于标准的平台，用以开发、部署和管理N层结构、面向Web的、以服务器为中心的企业级应用。
  • Java EE平台的主要目标是提供一个平台无关的、 基于组件的、可移植、多用户、安全和标准的企业级Java服务器端部署平台。
  • Java EE是一个规范，从实质上消除了对一家公 司的产品的依赖、并支持跨平台的应用部署。 Java EE提供了面向服务的体系结构，能够自动的对component进行支持和管理
  • 企业开发员将主要精力用于应用构件的开发，不必考虑底层的服务的实-
  • 将商务逻辑和服务分开，有助于重用

JavaEE 内容

  • 规范(Sepcification)
    • Java EE中的每个企业级API都有其自身的规范
  • 测试集(Test suites)
    • 应用于Java EE服务器的一套测试方法，对产品进行兼容性测试
  • 实现参考(Reference Implementation)
    • Java EE的实现参考
  • 设计蓝图文档(Blueprints document)
    • 企业级API在设计蓝图文档中都有定义
